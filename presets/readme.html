TrulyAmazed Level Format Specification

Level presets are stored in the JSON format as a list of objects, where each object is a level. Each level contains keys determining its settings, and the following keys are available:

Note: if a key is not given in a level, the generator will default to the value given in the editor. This is probably not what you want, so it's important to define keys evaluating to false instead of leaving them blank.

Note 2: When a level preset runs out of levels and no winning stage was specified, the last level defined in the levels list will be reused until the player runs out of fuel or otherwise loses. This is what allows Get to as many levels as possible!.json to generate mazes with the same settings indefinitely.

  • caption (string, optional) - Sets the caption that should display in the bottom of the game window. If not defined, defaults to the welcome caption specified in config.py
  • checkpoints (int) - Determines the amount of checkpoints present in the level.
  • darkness (boolean) - Determines whether darkness should be used
  • death_caption (string) - Determines what text should be displayed when the player loses a level. If not defined, defaults to the game over text defined in mazegame.py. This should usually contain one %s value for substituting the score in.
  • enemies (int) - Determines the amount of enemies present in level.
  • enemy_move_delay (int) - Determines the delay (in milliseconds) between each time enemies move. If set to 0, enemies will not move at all.
  • finish_bonus (int) - Determines the fuel bonus that the player should receive when they win a level.
  • flashlight_radius (int) - Determines the distance in tiles that the light around the player should illuminate when darkness is enabled. This defaults to half the maze width or height (whichever is smaller), rounded up. Obviously this has no effect when darkness is disabled.
  • fuel_pack_amount (int) - Determines the amount of fuel each fuel pack should give the player.
  • fuel_packs (int) - Determines the amount of fuel packs present in the level.
  • gunshot_fuel (int) - Determines the amount of fuel each gunshot (space key) takes up.
  • height (int) - Determines the height of the maze.
  • min_difficulty (int) - Determines the minimum difficulty, which is the minimum distance between the start and the finish points. This is incompatible with static starts/finishes and will be ignored if either is set.
  • starting_fuel (int) - Determines the amount of fuel the player should start the game. This ONLY takes effect when defined in the first level object of a level preset.
  • static_finish (two-length list of ints (e.g. [5,6]) - Determines the position of a static finish for the level. If this is outside the maze boundaries or not given, the setting will be ignored and a random finish will be generated.
  • static_start (two-length list of ints (e.g. [0,0]) - Determines the position of a static start for the level. If this is outside the maze boundaries or not given, the setting will be ignored and a start finish will be generated.
  • use_fuel (boolean) - Determines whether fuel should automatically decrease over time (as a time limit challenge).
  • width (int) - Determines the width of the maze.
  • win_caption (string) - Determines what text should be displayed when the player wins a level preset: i.e. when they beat a stage that has winning_stage set to True. If not defined, defaults to the game over text defined in mazegame.py. This should usually contain one %s value for substituting the score in.
  • winning_stage (boolean) - Determines whether the level in question should be a winning_stage: i.e. If they win, the game will stop with the win_caption specified.
